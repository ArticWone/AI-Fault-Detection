## What We Can Map Now

- Live HMI I/O point states for `DIO16 1` through `DIO16 4`.
- Drawing-level I/O names for at least the first two DIO16 pages from sheets 40 and 42.
- Several physical sensors/buttons to Modbus register candidates from tagged tests.

- Stopped versus running state with oven on: `10189` is the cleanest running value and `10059` is a strong stopped/running state value.

## What Still Needs Validation

- Exact HMI DIO point to Modbus register relationship. The HMI point names and Modbus register changes are both visible, but they need one-at-a-time toggles to bind confidently.
- B1/safety-bar signal is confirmed electrically as `DIO16 1 P13` (`E4.0.13`). The 2026-04-28 ON/OFF/ON Modbus retest did not produce a unique standalone register signature, so the Modbus exposure is likely encoded or grouped with the safety circuit.
- Some registers are packed or encoded state words, not one-register-one-sensor bits. For example, B2 and B12 show clean value changes, but not simple `0/1` boolean behavior.
- The front detection photoeye did not produce a clean standalone Modbus register in the 2026-04-28 clear/tripped/clear test. Best weak candidate is `10167`, which is already a broad process/safety/event value, so HMI DIO confirmation is needed.
- Auto/manual mode did not produce a clean standalone register in the current broad range (`10000:350`, `13300:500`). It may live outside the scanned range, be UI-only, or be encoded in a larger state structure.
- Wiring pages are scanned images, so drawing extraction is visual unless OCR is added later.
